## Задание 1. Типы данных, арифметика

**Исходные данные:** `prices = [19.99, 5.49, 3.50, 12.30, 49.64, 31.01, 7.99]`

### Ход вычислений

| Шаг | Формула |
|-----|---------|
| Сумма | `total = sum(prices)` |
| После скидки 7% | `after_discount = total × 0.93` |
| После НДС 20% | `after_vat = after_discount × 1.20` |
| Одна треть | `part = after_vat / 3` |

### Результаты

| Тип | Итоговая сумма | Одна часть (1/3) |
|-----|---------------|-----------------|
| `float` | ~135.599... | ~45.199... |
| `Decimal` | точно | точно |
| `Fraction` | точная дробь | бесконечная периодическая дробь |

### Вывод

- **float** хранит числа в двоичном формате IEEE 754. Многие десятичные дроби (например, 0.07) не представимы точно в двоичном виде, поэтому накапливается погрешность. Использовать для финансов **не рекомендуется**.
- **Decimal** хранит числа в десятичном представлении с произвольной точностью, поддерживает банковское округление. Является **стандартом для финансовых расчётов**.
- **Fraction** даёт математически абсолютно точный результат, но деление числителя на 3 при знаменателе, не кратном трём, даёт бесконечную периодическую дробь. При выводе необходимо округлять.

## Задание 2. Факториал

### Реализации

**Циклическая** — простой цикл `for i in range(2, n+1)`, умножает результат последовательно.

**Рекурсивная** — вызывает саму себя: `n * factorial(n-1)`, терминальный случай — `n == 0` или `n == 1`.

### Сравнение

| Критерий | Цикл | Рекурсия |
|----------|------|----------|
| Читаемость | Понятна новичкам | Элегантна математически |
| Память стека | O(1) | O(n) — по одному фрейму на вызов |
| Скорость | Быстрее | Медленнее из-за оверхеда вызовов |
| `factorial(1000)` | ✓ работает | ✗ `RecursionError` (лимит ~1000) |

### Вывод

При глубине рекурсии свыше ~900 Python выбрасывает `RecursionError` (лимит стека по умолчанию = 1000 фреймов). Для задач с большой глубиной рекурсии предпочтителен **итеративный** подход. Можно увеличить лимит через `sys.setrecursionlimit()`, но это не устраняет проблему, а лишь откладывает её.

## Задание 3. Подсчёт пробелов

### Структура рекурсивной функции

```python
def count_spaces_recursive(s: str, index: int = 0) -> int:
    if index == len(s):        # ← ТЕРМИНАЛЬНАЯ ЧАСТЬ (base case)
        return 0
    current = 1 if s[index] == ' ' else 0
    return current + count_spaces_recursive(s, index + 1)  # ← РЕКУРСИВНЫЙ ШАГ
```

- **Терминальная часть** — условие `index == len(s)`: когда все символы обработаны, возвращаем 0. Без неё рекурсия бесконечна.
- **Рекурсивный шаг** — вызов с `index + 1`: каждый вызов обрабатывает следующий символ.

### Вывод

| | Цикл | Рекурсия |
|-|------|----------|
| Читаемость | Проще | Сложнее для понимания |
| Память | O(1) | O(n) фреймов стека |
| Безопасность | Без ограничений | Риск `RecursionError` на длинных строках |

Для линейного обхода строки итерация предпочтительнее.

## Задание 4. Развёртывание списка

### Подходы

**Итеративно** — явный стек (`list`): извлекаем элементы, если элемент — список, помещаем его содержимое обратно в стек.

**Рекурсивно** — разбиваем на голову и хвост, рекурсивно разворачиваем оба.

### Вывод

- **Рекурсивный** вариант выразительнее, отражает структуру задачи.  
- **Итеративный** вариант безопаснее при глубокой вложенности (нет риска `RecursionError`) и быстрее из-за отсутствия оверхеда на фреймы стека.  
- Для производственного кода при неизвестной глубине вложенности рекомендуется **итеративный** подход.

## Задание 5. Быстрое возведение в степень

### Алгоритмы

| Алгоритм | Сложность | Принцип |
|---------|-----------|---------|
| Наивный (цикл) | O(n) | Умножать `base` ровно `n` раз |
| Быстрый (цикл) | O(log n) | Делить показатель пополам |
| Быстрый (рекурсия) | O(log n) | Делить показатель пополам рекурсивно |

**Идея «разделяй и властвуй»:** `base^n = (base^(n/2))^2` при чётном `n`, `base^n = base * base^(n-1)` при нечётном. Это сокращает число умножений с `n` до `log₂(n)`.

### Вывод

Алгоритм быстрого возведения в степень в сотни и тысячи раз быстрее наивного при больших показателях. Между итеративной и рекурсивной версиями быстрого алгоритма разница минимальна, итеративная чуть быстрее.

## Задание 6. Мемоизация

### 6.1 Декоратор `memoize`

Декоратор оборачивает функцию: при первом вызове с аргументами вычисляет и сохраняет результат в словарь (`cache[args]`), при повторном — возвращает из словаря. Подходит только для **чистых** (детерминированных) функций без побочных эффектов.

### 6.2 Пути в сетке m × n

Рекурсивная функция: `paths(m, n) = paths(m-1, n) + paths(m, n-1)`.  
Терминальный случай: `m == 1` или `n == 1` → один путь.  
Без мемоизации — экспоненциальная сложность из-за перекрывающихся подзадач. С мемоизацией — O(m×n).

### 6.3 Умножение на случайную величину

Рекурсивная функция, умножающая число на случайный коэффициент `depth` раз. **Мемоизация здесь неприменима** — функция не детерминирована: одни и те же аргументы возвращают разные результаты.

### 6.4 Проверка числа на простоту

Проверяет делители от 2 до √n рекурсивно.  
Терминальные случаи: `n < 2` → False; `divisor² > n` → True; `n % divisor == 0` → False.  
Рекурсивный шаг: `is_prime(n, divisor + 1)`.

## Ответы на контрольные вопросы

**1. Рекурсивная vs циклическая реализация?**  
Рекурсия вызывает функцию саму себя, создавая новый фрейм стека на каждый вызов. Цикл использует один и тот же блок памяти и не создаёт новых фреймов. Рекурсия лаконичнее для задач с естественной иерархической структурой.

**2. Что такое терминальная часть рекурсии?**  
Условие (base case), при котором рекурсия останавливается и возвращает базовое значение без нового рекурсивного вызова. Без неё рекурсия бесконечна и завершится `RecursionError`.

**3. Что происходит в памяти при рекурсии?**  
Каждый рекурсивный вызов создаёт новый фрейм в стеке вызовов, хранящий локальные переменные и адрес возврата. Память освобождается при возврате из вызова.

**4. Почему `factorial(1000)` рекурсивно вызывает ошибку?**  
Python по умолчанию ограничивает глубину стека ~1000 фреймами. Факториал 1000 требует ровно 1000 вложенных вызовов, что превышает (или достигает) лимит, вызывая `RecursionError`.

**5. В каких задачах рекурсия читаемее?**  
Обход деревьев и графов, быстрая сортировка, бинарный поиск, вычисление числовых последовательностей (Фибоначчи), парсинг вложенных структур данных.

**6. Идея «разделяй и властвуй»?**  
Задача разбивается на подзадачи меньшего размера, каждая решается независимо, результаты объединяются. Это снижает сложность алгоритма (например, с O(n) до O(log n)).

**7. Что такое мемоизация?**  
Оптимизация, при которой результаты предыдущих вызовов функции кэшируются по аргументам. При повторном вызове с теми же аргументами возвращается кэшированный результат вместо повторного вычисления.

**8. В каких задачах мемоизация даёт наибольший прирост?**  
В задачах с перекрывающимися подзадачами: числа Фибоначчи, динамическое программирование (пути в сетке, задача о рюкзаке), рекурсивный поиск с повторяющимися параметрами.

**9. Когда цикл предпочтительнее рекурсии?**  
При большой глубине итераций (риск `RecursionError`), при необходимости максимальной производительности, для простых линейных задач (сумма, поиск), когда важна экономия памяти стека.
