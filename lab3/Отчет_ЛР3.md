# Отчёт по Лабораторной работе №3: Динамическое программирование и жадные алгоритмы

## Задание 1. Теоретический анализ алгоритмов
В ходе выполнения работы были проанализированы следующие типы задач и методы их решения:

1. **Наибольшая сумма в подотрезке:** Используется **алгоритм Кадане** (один из видов динамического программирования).
2. **Непересекающиеся мероприятия:** Используется **жадный алгоритм** (сортировка по времени окончания).
3. **Размен монет:** Для произвольных наборов номиналов требуется **динамическое программирование**.
4. **Наибольшая общая подпоследовательность (НОП):** Классическая задача на **динамическое программирование** для двух строк.
5. **Заправки на маршруте:** Применяется **жадный алгоритм** (выбор самой дальней заправки).
6. **Рюкзак 0/1:** Решается методом **динамического программирования** (в нашем случае — с тремя ёмкостями).

## Задание 2. Максимальная сумма непрерывного подмассива (Алгоритм Кадане)
**Суть задачи:** Найти в массиве чисел такой непрерывный отрезок, сумма элементов которого была бы максимальной.

**Подход (Динамическое программирование):**
- **Состояние ДП:** `dp[i]` — максимальная сумма подмассива, который заканчивается ровно в элементе с индексом `i`.
- **Рекуррентное соотношение:** `dp[i] = max(A[i], dp[i-1] + A[i])`. 
  *Логика:* Либо мы "присоединяем" текущий элемент к предыдущему оптимальному подмассиву, либо начинаем новый подмассив с текущего элемента (если сумма с предыдущим отрицательна).
- **Сложность:**
    - По времени: O(N) — один проход по массиву.
    - По памяти: O(1) — достаточно хранить только текущую и глобальную максимальную сумму.

## Задание 3. Наибольшая общая подпоследовательность (НОП)
**Суть задачи:** Имеются две строки (например, зашифрованные сигналы). Нужно найти длину и саму подпоследовательность символов, которая содержится в обеих строках в том же порядке, но не обязательно непрерывно.

**Подход (Динамическое программирование):**
- **Состояние ДП:** `dp[i][j]` — длина НОП для префикса первой строки длины `i` и префикса второй строки длины `j`.
- **Переход:**
    - Если `A[i-1] == B[j-1]`, то `dp[i][j] = dp[i-1][j-1] + 1` (символ совпал, увеличиваем длину).
    - В противном случае: `dp[i][j] = max(dp[i-1][j], dp[i][j-1])` (выбираем лучший вариант из отбрасывания одного из текущих символов).
- **Сложность:** O(N * M) по времени и памяти, где N и M — длины строк.

## Задание 4. Минимальная стоимость пути
**Суть задачи:** В таблице чисел (матрице) найти путь от верхней левой клетки до нижней правой так, чтобы сумма чисел на пути была минимальной. Движение разрешено только вправо или вниз.

**Подход (Динамическое программирование):**
- **Состояние ДП:** `dp[i][j]` — минимальная возможная стоимость достижения клетки `(i, j)`.
- **Переход:** `dp[i][j] = cost[i][j] + min(dp[i-1][j], dp[i][j-1])`.
- **Почему жадный алгоритм не работает:** Если выбирать "самого дешевого соседа" на каждом шаге, можно попасть в "ловушку" — область с очень дорогими клетками, которые нельзя обойти. ДП же просматривает все пути "снизу вверх" и гарантирует глобальный минимум.
- **Сложность:** O(N * M) по времени и памяти.

## Задание 5. Задача о расписании (Выбор мероприятий)
**Суть задачи:** Дано множество мероприятий с временами начала и окончания. Нужно выбрать максимальное количество непересекающихся событий.

**Подход (Жадный алгоритм):**
- **Стратегия:** Сортируем мероприятия по **времени их окончания** (`end_time`).
- **Выбор:** На каждом шаге берем первое доступное мероприятие, которое начинается не раньше, чем закончилось предыдущее выбранное.
- **Обоснование:** Выбирая событие, которое заканчивается раньше всех, мы оставляем максимально возможное количество времени для всех последующих мероприятий.
- **Сложность:** O(N * log N) из-за сортировки.

## Задание 6. Минимизация остановок на заправку
**Суть задачи:** Маршрут длиной L. Бак позволяет проехать D. Нужно доехать до финиша, сделав минимум остановок на заправках.

**Подход (Жадный алгоритм):**
- **Стратегия:** Находясь в текущей точке, мы смотрим, какие заправки доступны в радиусе `D`. Из всех доступных выбираем ту, которая находится **дальше всего**.
- **Обоснование:** Каждая остановка "забирает" 1 балл. Максимизировав покрытое расстояние за одну остановку, мы минимизируем их общее количество.
- **Сложность:** O(N), где N — количество заправок.

## Задание 7. Размен монет
**Сравнение Жадного подхода и ДП:**
1. **Жадный алгоритм** (берем самый крупный номинал): Работает для "удобных" систем (1, 2, 5, 10, ...), но ошибается на нестандартных наборах (например, для суммы 6 при монетах 1, 3, 4 жадный выдаст 4+1+1 (3 шт), а ДП найдет 3+3 (2 шт)).
2. **Динамическое программирование:**
    - **Состояние:** `dp[w]` — минимальное число монет для суммы `w`.
    - **Переход:** `dp[w] = 1 + min(dp[w - coin])` для всех номиналов.
    - **Сложность:** O(Amount * N).

## Задание 7. Мульти-Рюкзак (Для трех кладоискателей)
**Суть задачи:** Есть набор предметов разного веса и стоимости. Есть три рюкзака с разной вместимостью. Нужно распределить предметы так, чтобы общая ценность была максимальной.

**Подход (Многомерное динамическое программирование):**
- **Состояние ДП:** `dp[i][w1][w2][w3]` — максимальная стоимость, которую можно получить из первых `i` предметов, распределив их по трем рюкзакам с оставшимися емкостями `w1, w2, w3`.
- **Переходы:** Для каждого предмета у нас есть 4 варианта:
    1. Не брать предмет вообще.
    2. Положить в 1-й рюкзак (если влезает).
    3. Положить во 2-й рюкзак (если влезает).
    4. Положить в 3-й рюкзак (если влезает).
- **Сложность:** O(N * C1 * C2 * C3), где C — вместимости рюкзаков. Это намного эффективнее полного перебора ($4^N$), хотя и требует много памяти.

## Контрольные вопросы
1. **Динамическое программирование (ДП)** — метод решения задач путем их разбиения на перекрывающиеся подзадачи, результаты которых сохраняются для повторного использования.
2. **Условия применимости ДП:** Оптимальная подструктура (оптимальное решение задачи строится из оптимальных решений подзадач) и наличие перекрывающихся подзадач.
3. **Состояние:** Уникальный набор параметров, описывающий текущую подзадачу (например, индекс и вес).
4. **Переход:** Логическая связь (формула), позволяющая вычислить ответ для текущего состояния, используя известные ответы для меньших подзадач.
5. **Мемоизация (Табуляция):** Процесс сохранения вычисленных значений в таблицу, чтобы избежать экспоненциального повторения вычислений.
6. **Восстановление ответа:** Процесс прохода по заполненной таблице ДП в обратном порядке (от финиша к старту) для выяснения, какие именно решения/предметы привели к итоговому оптимуму.
7. **Жадный алгоритм:** Алгоритм, на каждом шаге выбирающий локально лучший вариант без оглядки на будущее. Гарантирует успех только в задачах с определенной структурой (например, матроиды).
8. **Главное отличие:** Жадный алгоритм принимает одно окончательное решение на этапе. ДП рассматривает все варианты переходов и выбирает лучший на основе уже известных глобальных оптимумов.
9. **Почему жадный выбор в задаче о заправках оптимален?** Потому что выбор самой дальней доступной заправки всегда оставляет нам больше возможностей в будущем и не блокирует более оптимальные пути.
10. **Почему жадный выбор в задаче о рюкзаке 0/1 неверен?** Потому что предметы нельзя делить. Жадно взяв дорогой, но тяжелый предмет, мы можем занять место, в которое поместились бы два чуть менее дорогих, но легких предмета, дающих в сумме большую ценность.
