# Отчёт по реализации быстрого интерполяционного преобразования Хартли (БИПХ)

## 1. Постановка задачи

Требуется вычислить N-точечный спектр Хартли от сигнала, у которого заполнена только первая половина отсчётов (`x[0..N/2-1]`), а вторая половина равна нулю (`x[N/2..N-1] = 0`). Это называется **режимом половинной загрузки по входу** (half input loading).

Задача: реализовать это преобразование **быстро** — за `O(N log N)` операций вместо `O(N²)` для прямого матричного метода.

---

## 2. Математическая основа

### 2.1 Функция cas (cosine-and-sine)

Ядро дискретного преобразования Хартли — функция **cas**:

```
cas(θ) = cos(θ) + sin(θ)
```

Это вещественная альтернатива комплексной экспоненте Фурье `e^{jθ} = cos(θ) + j·sin(θ)`.

### 2.2 Дискретное преобразование Хартли (ДПХ)

Прямое N-точечное ДПХ определяется формулой:

```
X[k] = Σ_{n=0}^{N-1}  x[n] · cas(2π·n·k / N),    k = 0, 1, ..., N-1
```

В матричной форме:

```
X = H · x,    где  H[n,k] = cas(2π·n·k / N)
```

### 2.3 Обратное преобразование

При нормировке `Ho = H / √N` матрица ортонормальна: `Ho' · Ho = I`.
Это означает, что обратное преобразование — транспонирование той же матрицы:

```
x = Ho · X   (обратное ДПХ)
```

**Ключевое свойство ДПХ:** одна матрица используется и для прямого, и для обратного преобразования — в отличие от ДПФ, где нужны `H` и `H*`.

### 2.4 Интерполяционный случай (половинная загрузка)

Пусть входной сигнал имеет длину `N/2`, дополненный нулями до длины `N`:

```
x_pad[n] = x[n],   n = 0, ..., N/2 - 1
x_pad[n] = 0,      n = N/2, ..., N - 1
```

Тогда N-точечный спектр Хартли:

```
X[k] = Σ_{n=0}^{N/2-1}  x[n] · cas(2π·n·k / N)
```

Матрица преобразования становится **прямоугольной** размером `(N/2 × N)` — это интерполяционная матрица `Hi`. Результат — спектр длиной `N` из `N/2` входных отсчётов.

---

## 3. Описание алгоритма `fht_interp_half.m`

Функция реализует **быстрое интерполяционное ДПХ (БИПХ)** методом **DIF (Decimation In Frequency)** — прореживание по частоте с оптимизацией нулевой второй половины входа.

### Входные и выходные данные

```matlab
function X = fht_interp_half(x_in)
```

| Параметр | Тип           | Описание                                     |
|----------|---------------|----------------------------------------------|
| `x_in`   | вектор N/2×1  | Входной сигнал (только первая половина)      |
| `X`      | вектор N×1    | Спектр Хартли длиной N = 2·length(x\_in)    |

---

### Построчный разбор

#### Строки 13–14: Определение размеров

```matlab
N_half = length(x_in);
N = N_half * 2;
```

Вычисляем `N/2` и полный размер `N = 2·(N/2)`.

---

#### Строки 20: Оптимизация первого каскада (нулевая половина)

```matlab
X = [x_in, x_in];
```

**Математическое обоснование.** В первом каскаде DIF-алгоритма «бабочка» выполняет:

```
a = x[n],        b = x[n + N/2]
y = a + b
z = a - b
```

Поскольку вторая половина входа равна нулю (`b = 0`):

```
y = x[n] + 0 = x[n]
z = x[n] - 0 = x[n]
```

Обе ветви дают одинаковый результат — поэтому вместо вычислений просто **копируем** `x_in` дважды. Это исключает `N/2` сложений/вычитаний.

---

#### Строки 23–37: Поворотные множители первого каскада

```matlab
for i = 1:(N_half/2 - 1)
    m = N_half - i;
    idx1 = N_half + i + 1;
    idx2 = N_half + m + 1;

    a = X(idx1);
    b = X(idx2);

    angle = 2 * pi * i / N;
    c = cos(angle);
    s = sin(angle);

    X(idx1) = a * c + b * s;
    X(idx2) = a * s - b * c;
end
```

В первом каскаде DIF вторая половина рабочего буфера (`X[N/2..N-1]`) требует умножения на **поворотные множители** (twiddle factors). Используется формула Хартли-вращения для пары симметричных индексов `(i, N/2-i)`:

```
angle = 2π·i / N

X[N/2 + i]   = a · cos(angle) + b · sin(angle)
X[N/2 + m]   = a · sin(angle) - b · cos(angle)
```

где `m = N/2 - i` — симметричный индекс.
Индексы `i=0` и `i=N/4` (средняя точка) обрабатываются отдельно как частные случаи (тривиальные бабочки).

---

#### Строки 40–87: Оставшиеся каскады стандартного DIF

```matlab
L = N_half;
while L > 1
    L2 = L / 2;
    for block_start = 0:L:N-1
        % n=0: тривиальная бабочка
        a0 = X(block_start + 1);
        b0 = X(block_start + L2 + 1);
        X(block_start + 1)      = a0 + b0;
        X(block_start + L2 + 1) = a0 - b0;

        % Бабочки с поворотом
        for i = 1:(L2/2 - 1)
            ...
            angle = 2 * pi * i / L;
            X(idx_i)   = ai + bi;
            X(idx_m)   = am + bm;
            X(idx_i_b) = di * c + dm * s;
            X(idx_m_b) = di * s - dm * c;
        end

        % Средняя точка (i = L/4)
        ...
    end
    L = L2;
end
```

Это стандартный **DIF Хартли** для `log₂(N) - 1` оставшихся каскадов.
На каждом каскаде выполняется **бабочка Хартли** для пары элементов `(a, b)`:

```
верхний выход:  a + b
нижний выход:   (a - b) · cas(2π·i / L)
```

в раскрытом виде через пары симметричных индексов `(i, L/2-i)`:

```
X[i]   = a_i + b_i
X[m]   = a_m + b_m
X[i+L/2] = (a_i - b_i)·cos(angle) + (a_m - b_m)·sin(angle)
X[m+L/2] = (a_i - b_i)·sin(angle) - (a_m - b_m)·cos(angle)
```

Количество каскадов: `log₂(N) - 1` (первый каскад уже обработан отдельно).

---

#### Строка 90: Перестановка бит-реверс

```matlab
X = bitrevorder(X);
```

Стандартная финальная операция быстрых алгоритмов на основе DIF: переставляет элементы буфера в правильный порядок согласно инверсии двоичных индексов.

## 4. Структура алгоритма (схема)

```
Вход: x_in (N/2 отсчётов)
         │
         ▼
[Шаг 1] Копирование: X = [x_in | x_in]   ← оптимизация нулевой половины
         │
         ▼
[Шаг 2] Поворотные множители 2-го полублока (каскад 1)
         │
         ▼
[Шаг 3..log₂N] Стандартные каскады DIF Хартли
         │
         ▼
[Шаг log₂N+1] Bit-reversal перестановка
         │
         ▼
Выход: X (N отсчётов спектра Хартли)
```

## 5. Сравнение сложности: полный алгоритм vs. половинная загрузка

| Метод                        | Сложность           | Операций при N=1024 |
|------------------------------|---------------------|---------------------|
| Прямой матричный (O(N²))     | `N²` умножений      | ~1 048 576          |
| Полный БПХ (O(N log N))      | `N/2 · log₂(N)`     | ~5 120              |
| **БИПХ — половинная загрузка** | `N/2 · log₂(N) - N/2` | **~4 608**       |

**Выигрыш БИПХ перед полным БПХ:**

```
ΔOp = N/2  сложений/вычитаний  (исключён весь первый каскад)
```

При `N = 1024`: экономия `512` операций сложения (~10% от полного БПХ).

**Выигрыш БИПХ перед прямым матричным методом:**

```
Ускорение ≈ N / log₂(N) = 1024 / 10 ≈ 102 раза
```

Это подтверждает и тест в `test_fht.m` (строка 49):

```matlab
fprintf('Speedup: %.1f x\n', t_direct / t_fast);
```

## 6. Преимущества алгоритма

1. **Вещественная арифметика.** ДПХ использует только действительные числа — нет накладных расходов на комплексное умножение, как в ДПФ.

2. **Одна матрица — прямое и обратное преобразование.** Свойство `Ho' = Ho⁻¹` позволяет использовать одну функцию для обеих операций.

3. **Оптимизация нулевой половины.** Вместо вычисления `N/2` бабочек первого каскада — одно копирование массива. Это исключает `N/2` умножений и `N/2` сложений.

4. **Симметрия бабочек.** Обработка пар `(i, L/2-i)` позволяет вычислять два выходных значения за одно тригонометрическое вычисление (одна пара `cos/sin` на оба элемента).

5. **Асимптотическая сложность `O(N log N)`** против `O(N²)` для прямого метода — при `N = 1024` ускорение более **100 раз**.
