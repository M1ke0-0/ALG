## ⚡ Быстрая сводка

| Алгоритм            | Сложность    | Отсортирован? | Когда использовать              |
|:--------------------|:------------:|:-------------:|:--------------------------------|
| Линейный (барьер)   | O(n)         | ❌ Нет        | Маленькие/несортированные данные |
| Бинарный            | O(log n)     | ✅ Да         | Стандартный выбор для сорт. данных |
| Интерполяционный    | O(log log n) | ✅ Да         | Равномерно распределённые данные |
| Экспоненциальный    | O(log n)     | ✅ Да         | Большие/неограниченные массивы   |

---

## 🔍 1. Линейный поиск с барьером

**Идея:** добавляем `target` в конец → обход без проверки `i < n` на каждом шаге.

```python
def linear_search_barrier(arr, target):
    n = len(arr)
    arr_copy = arr + [target]   # ← барьер
    i = 0
    while arr_copy[i] != target:
        i += 1
    return i if i < n else -1  # -1 если нашли только барьер
```

**Сложность:** O(n) · **На любых данных**  
**Барьер даёт:** убирает одно сравнение `i < n` из цикла → чуть быстрее.

---

## 🔍 2. Бинарный поиск

**Идея:** каждый шаг делим пространство поиска пополам.

```python
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2          # середина
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1              # идём вправо
        else:
            hi = mid - 1             # идём влево
    return -1
```

**Сложность:** O(log n) · **Только для отсортированного!**  
**Пример:** N=1000 → максимум **10** сравнений (log₂1000 ≈ 10)

---

## 🔍 3. Интерполяционный поиск

**Идея:** предсказываем позицию, как в телефонной книге — «А» ищем в начале, «Я» — в конце.

```python
def interpolation_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi and arr[lo] <= target <= arr[hi]:
        if arr[lo] == arr[hi]:
            return lo if arr[lo] == target else -1
        # Формула интерполяции:
        pos = lo + (target - arr[lo]) * (hi - lo) // (arr[hi] - arr[lo])
        pos = max(lo, min(hi, pos))   # не выходим за границы
        if arr[pos] == target:
            return pos
        elif arr[pos] < target:
            lo = pos + 1
        else:
            hi = pos - 1
    return -1
```

**Сложность:** O(log log n) при равномерных / O(n) в худшем случае  

**Формула интерполяции:**
```
pos = lo + (target - arr[lo]) × (hi - lo) / (arr[hi] - arr[lo])
```

> ⚠️ На равномерных данных — лучший. На квадратичных/экспоненциальных — хуже бинарного!

---

## 🔍 4. Экспоненциальный поиск

**Идея:** сначала экспоненциально (1→2→4→8→...) ищем диапазон, потом бинарный поиск в нём.

```python
def exponential_search(arr, target):
    n = len(arr)
    if arr[0] == target:
        return 0
    # Шаг 1: расширяем диапазон
    i = 1
    while i < n and arr[i] <= target:
        i *= 2                        # i = 1, 2, 4, 8, 16...
    # Шаг 2: бинарный поиск в [i//2, min(i, n-1)]
    lo, hi = i // 2, min(i, n - 1)
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1
```

**Сложность:** O(log n) · **Преимущество:** работает на массивах неизвестной/бесконечной длины.

## 📊 Когда что выбирать

```
Данные не отсортированы?
  └─► Линейный поиск
  
Одиночные запросы к отсортированному массиву?
  └─► Бинарный поиск
  
Много запросов к несортированному массиву?
  └─► Sorted() + Бинарный (выгодно при K ≥ ~6 запросов на N=10K)
  
Равномерно распределённые данные?
  └─► Интерполяционный (быстрее в ~5 раз!)
  
Очень большой / неограниченный массив?
  └─► Экспоненциальный
  
Нужна максимальная скорость в продакшне?
  └─► numpy.searchsorted / bisect.bisect_left
```

## 🔑 Ключевые формулы

| Понятие | Формула |
|:--------|:--------|
| Бинарный поиск: середина | `mid = (lo + hi) // 2` |
| Интерполяционный: позиция | `pos = lo + (x - a[lo]) * (hi - lo) // (a[hi] - a[lo])` |
| Макс. итераций бинарного | `⌈log₂(N)⌉` |
| Порог Sort+Binary | `K ≥ T_sort / (T_linear - T_binary)` |

## 💡 Ответы на контрольные вопросы (кратко)

| Вопрос | Ответ |
|:-------|:------|
| Что такое линейный поиск? | Перебор всех элементов по порядку, O(n) |
| Что такое бинарный поиск? | Деление пополам, O(log n), только для сорт. |
| Зачем сортировка для бинарного? | Решение «вправо/влево» работает только в упорядоченном массиве |
| Что даёт барьер? | Убирает проверку `i<n` из внутреннего цикла |
| Идея интерполяционного? | Предсказание позиции по соотношению значений |
| Почему хорош на равномерных? | Формула точно угадывает позицию за 1 шаг |
| Идея экспоненциального? | Удваивает диапазон, затем бинарный поиск |
| Когда сортировать перед поиском? | При K ≥ ~6–20 запросов (зависит от N) |
| Зачем считать сравнения, не только время? | Машинно-независимая метрика, не зависит от CPU/ОС |
| Отличие своей реализации от библиотечной? | Библиотечные (NumPy, bisect) — на C, в 10–100× быстрее |

## 🐍 Полезные библиотечные аналоги в Python

```python
import bisect

arr = sorted([5, 1, 3, 8, 2])  # [1, 2, 3, 5, 8]

# Найти позицию вставки (бинарный поиск):
pos = bisect.bisect_left(arr, 3)  # → 2

# Проверить наличие:
found = arr[pos] == 3             # True

# NumPy:
import numpy as np
np_arr = np.array(arr)
idx = np.searchsorted(np_arr, 3)  # → 2
```
