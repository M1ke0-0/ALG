## Задание 1. Реализация алгоритмов поиска

### Реализованные алгоритмы

#### 1. Модифицированный линейный поиск (с барьерным элементом)

Перед поиском искомое значение добавляется в конец массива (барьер). Это исключает проверку выхода за границу на каждой итерации — цикл гарантированно завершится. После нахождения проверяется, достигли ли реального конца.

```python
def linear_search_barrier(arr, target):
    n = len(arr)
    arr_copy = arr + [target]   # барьерный элемент
    i = 0
    while arr_copy[i] != target:
        i += 1
    return i if i < n else -1
```

**Сложность:** O(n) | **Применим:** к любым массивам

---

#### 2. Бинарный поиск

На каждом шаге область поиска делится пополам. Сравнивается средний элемент с искомым; в зависимости от результата — левая или правая половина отбрасывается.

```python
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1
```

**Сложность:** O(log n) | **Требует:** отсортированного массива

---

#### 3. Интерполяционный поиск

Вместо середины вычисляет предполагаемую позицию по формуле линейной интерполяции:

```
pos = lo + (target - arr[lo]) * (hi - lo) / (arr[hi] - arr[lo])
```

**Сложность:** O(log log n) при равномерных данных, O(n) в худшем случае  
**Требует:** отсортированного массива

---

#### 4. Экспоненциальный поиск

Шаг 1 — экспоненциально удваивает диапазон (1, 2, 4, 8, ...) пока `arr[i] ≤ target`.  
Шаг 2 — выполняет бинарный поиск в найденном диапазоне `[i/2, i]`.

**Сложность:** O(log n) | **Требует:** отсортированного массива  
**Преимущество:** хорош для больших/неограниченных массивов

## Задание 3. Порог выгодности «Sort + Binary»

### Условие

Имеется неотсортированный массив N = 10 000. Сравниваем:
- **Стратегия A:** K линейных поисков в неотсортированном массиве
- **Стратегия B:** однократная сортировка + K бинарных поисков

## Контрольные вопросы

**1. Что такое линейный поиск?**  
Последовательный перебор всех элементов массива до нахождения искомого. Работает за O(n), применим к любым данным.

**2. Что такое бинарный поиск?**  
Алгоритм поиска в отсортированном массиве: на каждом шаге область делится пополам. Работает за O(log n).

**3. Почему бинарный поиск требует сортировки?**  
Решение о сужении области основано на сравнении с серединным элементом. Без упорядоченности невозможно знать, в какой половине находится искомый элемент.

**4. Что даёт барьер в модифицированном линейном поиске?**  
Барьер — искомый элемент, добавленный в конец массива. Цикл гарантированно завершится без проверки границы `i < n` на каждом шаге, что ускоряет выполнение.

**5. В чём идея интерполяционного поиска?**  
Вместо середины вычисляется предполагаемая позиция по формуле линейной интерполяции — «угадывается», где примерно находится элемент по его значению.

**6. Почему он хорошо работает на равномерных данных?**  
На равномерных данных соотношение значений пропорционально соотношению индексов — формула интерполяции точно предсказывает позицию с первой попытки.

**7. В чём идея экспоненциального поиска?**  
Сначала удваивается диапазон (1, 2, 4, 8, ...) до первого элемента ≥ target. Затем бинарный поиск в найденном диапазоне. Быстро сужает область при неизвестной длине массива.

**8. Когда выгодно сортировать массив перед поиском?**  
При числе запросов K ≥ порогового значения (≈6 для N=10 000). Стоимость однократной сортировки окупается экономией на K быстрых бинарных поисках.

**9. Почему нужно считать не только время, но и число сравнений?**  
Время зависит от CPU, ОС, кэша. Число сравнений — машинно-независимая «чистая» метрика, позволяющая честно сравнивать алгоритмы.

**10. Чем пользовательская реализация отличается от библиотечной?**  
Библиотечные функции (NumPy) реализованы на C, используют оптимизацию компилятора и SIMD-инструкции. Пользовательская реализация на Python интерпретируется → в 10–100 раз медленнее при больших N.
